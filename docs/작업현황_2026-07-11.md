## 재시작 시 주의

- test는 확정 config로 1회 평가 원칙 — **재실행 불필요, 하지 말 것.** 모든 튜닝·실험은 dev 전용.
- 캡션 수정은 `common.is_corrupted_caption` 자동 판정분만(`m3 --recaption-corrupted`), 수동 편집 금지.
- 재캡셔닝 후 m4 재실행 필수 아님 — text_hash가 자동 감지(단, m4는 돌려야 함. M5가 불일치 시
  ValueError로 차단하므로 무증상 낡은 임베딩은 없음).
- ablation 변형(`work_p1~p3`, `work_seg3/10`, `config_p*/seg*.yaml`)은 dev 검증 완료된 산출물 —
  결론이 문서화됐으므로 삭제 가능하나 발표 전까지 보존 권장. `work_bge`, `work_rp13`은 폐기된
  구 변형(오염 잔존 있음, 현행 결론과 무관).
- dev-ablation 변형 3곳에 오염 캡션 각 1건 잔존(재시도 실패 시 greedy 유지 규약) — 정상,
  재작업 불필요. 상세: DESIGN_SPEC 8-5(4).
