# 영상 장면 검색 파이프라인 — 진행 보고 (2026-07-09, 3주차)

한국어 영상에서 자연어 질의로 장면을 찾고, 근거 인용이 강제된 AAR 리포트를 자동 생성하는
시스템. 자막 단독 검색(baseline, α=1.0)과 자막+VLM 장면설명 결합 검색(proposed)의 비교가
연구의 축. — 정대석 · KAIST 김주호 교수님 연구실

## 핵심 지표

- **M1~M9 전 모듈 + 웹 UI 구현 완료** — 계획 대비 약 6주 선행 (3주차에 9주차분)
- 자동 테스트 **96개 전부 통과**, 경고 0 (master `8022e8e`)
- 평가 질의 라벨 **18/60** — 유일한 병목, 현재 보류 결정(평가 신뢰성 목적이지 성능 향상 수단 아님)

## 파이프라인 구조

M1 ffmpeg 5초 분할 → M2 대표 프레임(순차 디코딩) → M3 Whisper 자막 + Qwen2.5-VL 캡션
→ M4 KURE-v1 임베딩 → M5 α 결합 검색 → M6 Hit@k·MRR 평가.
별도로 M7-W 웹 UI(업로드→자동 인덱싱→채팅 검색→구간 점프), M8 AAR 리포트([seg#N] 인용 강제),
M9 G-Eval judge 평가.

원칙: 전 모듈 파일(JSON/NPY) 통신의 독립 CLI · 멱등 실행 · 온프레미스(클라우드 API 없음) ·
baseline은 동일 코드 경로의 α=1.0 특수 경우(비교 대칭성).

폐기 전에는 격차가 과장돼 있었다 — 라벨 품질이 결론을 좌우함을 확인. 정식 수치는 60질의 후.

### 이번 주 실측 확정 사항

1. **캡션 문자혼입 원인** — repetition_penalty 1.3이 3B-4bit VLM에서 한자·가나 혼입 유발
   (표본 10개 중 8→3개로 감소 실측) → config 1.1 확정, 재캡션·재인덱싱 완료.
2. **M2 인덱싱 병목 제거** — 세그먼트별 랜덤 시크 ~5,000회가 지배 비용. 순차 디코딩 전환으로
   샘플링 3.6× 단축(25.3s→7.0s), 26분 영상 ~2×(추정). 대표 프레임 선택은 두 영상 645세그먼트에서
   기존과 완전 일치(motion_score |Δ|<1e-6) 검증 후에만 머지 — 재캡션 불필요.
3. **static_threshold 스윕** — 0.08~0.10 구간에서 장면형 성능 급락(hit@5 0.75→0).
   현행 0.05는 안전 구간 → 유지, 60질의 후 재실측.
4. **무관련 질의 문제** — per-video min-max 정규화가 "관련 없음" 신호를 소거, 무관 질의도
   0.9대 확신 점수(유관 raw 코사인 ≈0.62 vs 무관 ≈0.47). raw 통계 로깅 가동, 임계값은 데이터 축적 후.
5. **α* 소표본 불안정** — dev 12질의로는 α*가 캡션 버전에 따라 0.5↔0.3 요동
   → 쌍체 부트스트랩 안정화 구현(아래).
