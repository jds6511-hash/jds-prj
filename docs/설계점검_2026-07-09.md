## B. 설계 개선 기회 (우선순위)

### HIGH-1. M2 디코딩 경로 교체 — 최대 속도 레버
현행 m2_keyframe.py는 프레임마다 랜덤 시크(26분 영상 ≈ **5,000회 seek**) →
M2 50분의 정체. 순차 1회 통과 디코딩으로 교체(+대표 프레임만 seek 312회).
효과: M2 수 배 단축, ablation 실험 1(최고 비용) 직접 절감. 재인덱싱 불필요
(대표 프레임 선택 보존 검증 전제). 난이도 낮음, GPU 0.

### HIGH-2. 원본 코사인 로깅 — 무관련 판정 근거 축적
m5_search가 정규화 이전 raw 코사인(max/mean)을 함께 반환·로깅.
랭킹 불변·지표 무영향. "에어컨수리" 문제(무관련도 0.9대 확신)의 판정 임계값을
설계할 데이터가 쌓이기 시작. 난이도 최하, GPU 0. **가장 값싼 다음 행동.**

### HIGH-3. α 안정화 — 헤드라인 주장의 신뢰성
실측: dev 12개(영상 1)로는 α*가 캡션 버전 따라 0.5↔0.3 요동, test 전이 나쁨.
(a) dev 영상 2~3개로 다양화, (b) 지표에 부트스트랩 신뢰구간 병기,
(c) alpha_select_metric을 hit@5 단독 → MRR(또는 병합)로 평활화.
(b)(c)는 난이도 낮음·GPU 0, (a)는 라벨링·인덱싱 비용(HIGH-1 후 완화).

### MEDIUM-4. 캡션 max_new_tokens 상향+문장완결 후처리 → ablation 실험 3의 전제조건
128 하드코딩(config에도 없음), 캡션 29% 잘림 의심. 실험 3의 상세형 프롬프트가
이 캡에서 confound됨. config화 + 192~256 상향 + 미완결 문장 절단. 실험 3에 흡수.

### MEDIUM-5. 문자혼입 후처리 정규화
잔여 혼입(카모フラ주제 등 모델 어휘)은 CPU 정규화 테이블로 교정.
재캡션 불필요, M4 재임베딩(분 단위)만.

### MEDIUM-6. 결합 임베딩(emb_joint) 제3 arm
가이드 3장 원안(자막+캡션 블록 통째 임베딩) vs 현행 분리+α — 선택의 정량 근거
확보용 실험. M4 임베딩 1 pass 추가. 시간 여유 시.

### 기각: 질의 확장(근거 없음), threshold 확대(실험 2가 커버), 화면텍스트 금지
프롬프트(실험 3에 흡수), M2 저해상도 차분(병목은 seek), 다중 프레임 캡션(실험 1
결과 대기), KURE prefix(해당 없음).
