## 1. 문제 발견: per-query 순위 분해

dev 96건 확정 시점(커밋 `e3d7a61`)의 test 평가(α*=0.6)는 집계상 mrr 0.7135→0.7417로
개선처럼 보였으나, per-query 순위를 분해하면 다른 그림이 나왔다.

- test 19건 중 **12건은 baseline/proposed 모두 rank 1**(포화) — 자막에 답이 그대로 있어
  두 방법을 전혀 구분하지 못하는 질의.
- 실제 경합은 7건뿐이고, 그중 **pb_q08(복합형)만 유일하게 회귀**(rank 1→2).
- 장면형은 큰 폭의 순위 개선(204→16, 196→31, 116→19, 37→3)에도 대부분 top-10 밖.

## 2. pb_q08 회귀의 근본 원인: VLM 캡션 오염

pb_q08("새해 카운트다운 후 불꽃놀이를 보며 새해 인사를 나누는 장면")의 정답 세그먼트
245·246의 캡션이 **전부 중국어로 생성**되어 있었다(Qwen2.5-VL-3B 4bit의 언어 이탈 —
`common.is_corrupted_caption`이 잡는 유형 그대로). 자막("새해 복 많이 받으십쇼")은 정답과
정확히 일치해 baseline은 1위였지만, proposed는 오염 캡션 임베딩이 (1−α) 가중치로 섞이며
2위로 밀렸다. **회귀의 원인은 결합 방식이 아니라 캡션 데이터 품질이었다.**

전수 스캔 결과 오염 캡션은 5개 영상 1,039세그먼트 중 21건:
panibottle 17(idx 102–125, 238–248 구간 집중), Wilderness 3(idx 3, 4, 134), gwaktube 1(idx 38).

## 3. 조치: 오염 캡션 선별 재생성 (내용 중립적 데이터 품질 수정)

- `m3_generate.py --recaption-corrupted`: `is_corrupted_caption` 감지분만 비우고 재생성.
- greedy 디코딩은 결정적이라 단순 재실행은 같은 오염 출력을 재현 → 오염 감지 시
  **샘플링(temperature 0.7) 재시도 최대 2회**로 전환(`caption_all` 공통 로직).
- **방법론 노트:** 대상 선정은 자동 판정 함수로만 이뤄졌고(질의 정답 여부와 무관하게 전
  영상 일괄 적용), 캡션 내용을 수동 편집하지 않았다. test 질의를 보고 캡션을 고치는
  경로가 아니므로 test 오염(leakage)에 해당하지 않는다. 다만 test 영상의 입력 데이터가
  바뀌었으므로 전후 수치를 모두 기록한다(아래 5절).
- 결과: 21건 중 20건 정상화. **panibottle idx 112는 재시도 후에도 중국어 잔존**(잔여
  오염 1건, 어떤 질의의 정답 세그먼트도 아님 — M8/M9는 `is_corrupted_caption` 필터로 방어).

## 4. 재캡셔닝 후 dev α 재탐색 (dev 96건)

재생성된 캡션이 dev 영상(Wilderness, gwaktube)에도 포함되므로 규정대로 α 재탐색부터 수행.

- alpha_best_point=0.4 (유지), **alpha_star=0.5** (0.6→0.5), tie_set=[0.0–0.5] 6개.
- 직전(재캡셔닝 전, dev 96)에는 tie_set=[0.0–0.6] 7개였다. α=0.6의 차이 CI가
  [-0.1095, -0.0037]로 0을 배제하게 되어 tie_set에서 탈락 — 캡션 품질 개선이 저α 구간의
  우위를 더 선명하게 만든 방향의 변화다.

## 5. test 평가 전후 비교 (n=19, baseline=α1.0)

| 지표 | 재캡셔닝 전 (α*=0.6) |  | 재캡셔닝 후 (α*=0.5) |  |
|---|---|---|---|---|
| | baseline | proposed | baseline | proposed |
| hit@1 | 0.6842 | 0.6842 | 0.6842 | **0.7368** |
| hit@5 | 0.7368 | 0.7895 | 0.7368 | **0.9474** |

- **proposed−baseline 쌍체 부트스트랩 95% CI (신규 지표):** mrr [0.0304, 0.1843],
  hit@5 [0.0526, 0.4211] — **모두 0을 배제**, 단발 점추정이 아닌 구간으로도 개선이 지지됨.
- **경합/포화 분리 (신규 지표):** 포화 13 / 경합 6. 경합 6건만의 지표:
  baseline mrr 0.0929·hit@5 0.1667 vs proposed mrr 0.4167·hit@5 0.8333.
- per-query: pb_q08 회귀 해소(1→1). 장면형 순위 204→4, 196→12, 116→3, 37→2 (+복합형 86→3).
  경합 6건 중 5건이 top-5 진입(재캡셔닝 전에는 1건).

## 6. 남는 한계 (발표 시 명시할 것)

1. **test 표본이 작다**: 질의 19건, 영상 2개. 경합 부분집합은 6건. by_type 수치(특히
   장면형 n=5)는 질의 1건 변동에 ±0.2씩 움직인다. CI를 병기하는 이유.
2. **포화 12~13건은 방법 비교에 기여하지 않는다**: 헤드라인 mrr 0.82는 자막만으로 풀리는
   쉬운 질의가 깔아준 값. 방법 간 차이의 실체는 경합 6건의 분리 표로 봐야 한다.
3. **gemini_promo는 번인 자막이 있는 홍보 영상**으로 일반 사용 시나리오와 거리가 있음
   (자연스러운 test 영상은 panibottle 1개).
4. **hit@1 개선(0.68→0.74)은 질의 1건 차이**: pb_q09(2→1). 일반화 주장에 쓰지 말 것.
5. **VLM 캡션 오염이 재발 가능**: 샘플링 재시도로도 못 잡은 잔여 1건(idx 112) 존재.
   3B-4bit 모델의 구조적 한계로, 캡셔너 업그레이드가 향후 과제.
