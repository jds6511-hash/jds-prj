# 중간발표 예상질문 방어 스크립트 (2026-07-13 확정)

> 원칙: 먼저 인정하고, 실측 근거로 되받는다. 모든 수치는 DESIGN_SPEC 8-6과
> ablation_plan 재검증 표(1-6-1, 3-6-1) 기준. 숨기면 잡히고, 선제하면 강점이 된다.

## Q1. "장면형에서 이긴 건 당연한 것 아닌가? baseline은 자막만 보는데."

**인정**: 맞다. baseline(α=1.0, 자막 전용)은 무발화 장면을 구조적으로 못 찾는다.
장면형 개선(mrr 0.174→0.667)은 놀라운 발견이 아니라 **예견된 사각지대의 정량화**다.

**되받기**: 이 연구의 주장은 "이겼다"가 아니라 세 가지다.
① 그 사각지대가 실사용 질의의 1/3(장면형 13/39)이라는 규모의 정량화,
② 캡션을 더해도 **자막이 강한 유형을 거의 해치지 않는다**는 트레이드오프의 실측
   (복합형 0.825→0.840, 자막형 0.958→0.882로 소폭 하락 — 이게 진짜 비자명한 부분),
③ 그 균형점(α*)을 소표본에서도 재현 가능하게 찾는 절차(부트스트랩 tie_set).
발표 무게는 ②③과 엔지니어링(abstention·환각 제어·재현성 규율)에 둔다.

## Q2. "hit@5, hit@10은 유의하지 않은데 개선이라 할 수 있나?"

**인정**: hit@5/10의 95% CI [-0.026, 0.282]는 0을 포함한다 — 유의 주장 안 한다(문서에 명시).

**되받기**: mrr [0.042, 0.257]과 hit@1 [0.026, 0.282]은 0을 배제한다. n=39의 검정력
한계는 정직하게 보고하며, 유의한 지표와 아닌 지표를 구분해서 쓰는 것 자체가 이
프로젝트의 방법론이다. 포화 분석(21건 양측 1위)과 경합 18건 분해(mrr 0.239→0.556)로
"어디서 개선이 나오는지"까지 보였다.

## Q3. "test를 여러 번 돌린 것 아닌가?"

**인정**: 공식 평가 실행은 총 4회다(n=19 초기 → n=39 확장 → thr=0 확정 → 세정 재검증).

**되받기**: 정확한 원칙은 "**튜닝 목적 접촉 0회**"다. 매 실행은 dev에서 결정이 끝난 뒤의
확인이었고 test 결과가 config 선택에 역류한 적 없다(8-6에 실행별 사유 명기). 경계 사례
1건(pb_q08 원인 규명이 test 관찰에서 출발)도 처치는 내용 무관 자동 판정 기준의 전 영상
일괄 적용이었음을 문서화했다 — 이 수준까지 접촉 이력을 공개하는 연구는 드물다.

## Q4. "IoU 지표가 hit@1과 똑같은데?"

**인정**: 맞다. GT가 5초 격자 정렬 + test GT 전부 1~2세그 조건에서 iou@0.5_r@1 ≡ hit@1
(수학적 항등, 8-6 각주). 독립 지표로 세지 않는다.

**되받기**: IoU가 분별력을 갖는 곳은 seg_len ablation(격자 불일치 발생)이고, 거기서
실제로 사용했다(1-6: 3초 세그의 IoU 상한 0.6 문제 포함 격자 종속성 분석).

## Q5. "α=0.5는 어떻게 정했나? 자의적이지 않나?"

dev 96건 grid search에서 MRR 선택 + 점추정 1위(α=0.4) 기준 쌍체 부트스트랩 CI로
통계적 동률 집합(tie_set [0.2–0.5])을 구하고, 동률 내 자막 우선 tiebreak로 0.5 선택.
**보수적 선택이라 보고된 개선치는 하한이다** — 점추정 최적 α=0.4였다면 더 컸다.
영상별 최적(0.3/0.4/0.5)도 전부 tie_set 안(단일 α의 방어 근거, by_video 로깅).

## Q6. "VLM 캡션에 중국어가 섞인다던데 평가를 오염시키지 않나?"

**인정**: Qwen2.5-VL-3B-4bit의 언어 드리프트는 실재하고(~1% 세그먼트), 재시도로도
일부 잔존한다. 근본 해소는 서버 GPU(더 큰 모델/고비트)의 향후 과제다.

**되받기**: ① 세정은 자동 판정 함수 기준만 사용(내용 편집형 leakage 아님, 1,587건
오탐 0 검증), ② harm의 실측: 오염 캡션은 타 질의 랭킹에 무해(false positive 0/8,
전부 31~377위로 침강)하고 해당 세그먼트 자신만 못 찾게 되는 국소 손실(~0.2 코사인,
docs/probes/corrupt_harm_probe.py로 재현 가능), ③ 재세정 후 test 순위 39건 전건
불변으로 평가 오염 없음을 확인했다.

## Q7. "정적 치환(static_threshold)은 왜 껐나? 환각 방어라며?"

이 프로젝트에서 가장 좋은 발견 서사다: 치환 트리거(motion_score 낮음)가 **정확히
proposed가 이겨야 할 케이스**(정적 화면에 시각 정보, 무발화)와 겹친다는 설계 긴장을
ablation으로 실증했다 — kheritage 다큐에서 치환 해제 시 kh_q17 115→7위 등, thr=0이
유의 우세(CI 0 배제). "안전장치가 핵심 타겟과 충돌함을 발견하고 끈 것"(2-4-3).
환각 방어는 치환 대신 오염 캡션 선별 재생성(8-5(4))이 담당한다.

## Q8. "무관 질의가 들어오면?"

abstention: max(sub, cap) 채널 τ=0.55, dev 96 vs 무관 20으로 캘리브레이션 —
오배제 0/96, 무관 감지 70%. 결과는 은폐하지 않고 경고 배너만(연구 도구로서 오판
관찰 필요). 초기 sub 단독 채널이 장면형 유관 질의를 오배제하는 편향을 발견해 개정한
과정 자체가 "캡션 채널이 검색뿐 아니라 abstention도 개선"의 근거. 한계: τ 여유 0.008,
"자명히 무관" 질의만으로 캘리브레이션한 selection bias(8-2 명시).

## Q9. "M8(리포트)/M9(judge)는 왜 결과가 없나?"

**인정**: 실구동 전이다. 7B가 로컬 6GB에 물리적으로 안 들어가고(4bit로도 비양자화
embed·lm_head 초과 실측), 3B 하향은 프롬프트 예시 복사 오염으로 기각했다.

**되받기**: 설계·구현·단위테스트 21개 완료 + **negative control 통과**(M9 judge가
오염 리포트를 정확히 groundedness 0으로 판정). "안 돌려봤다"가 아니라 "돌릴 자격이
있는 코드가 서버 GPU를 기다린다"는 상태. 인용 강제·부분집합 검증·교차 세그먼트 추론
판정 기준(4-9)까지 확정돼 있다.

## Q10. "프롬프트 ablation에서 P0가 이긴 건 선택 편향 아닌가?"

**인정**: dev 질의 라벨링 때 후보 스캔에 P0 캡션이 참조됐다 — "P0 우세의 확증"이 아니라
"**후보들이 P0를 이기지 못함**"으로 읽는 게 맞다(3-6 정직 보고 그대로).

**되받기**: threshold confound를 발견하고 동일 조건(thr=0, 동일 세정)으로 재검증까지
했고 격차는 오히려 확대됐다(3-6-1). 캡션 단독(α=0) 원신호에서 격차가 가장 크다.

## Q11. "검색이 단일 영상 안에서만 되는데?"

맞다 — 스코프가 "영상 1개 내 모먼트 검색"(문제 정의)이다. 교차 영상 검색은 인덱스
구조(per-video minmax가 교차 영상 비교를 깨는 지점 포함)의 재설계가 필요한 확장 과제로
분리했다.

## Q12. "세그먼트 5초는 왜?"

EventFormer 차용값이었으나 본 데이터에서 자체 재확인했다(1-6-1 재검증: 3/5/10초 중
5초가 MRR·hit@5·장면형 전부 최고). 부수 발견: 세그먼트가 길수록 자막 문맥이 온전해져
자막 모달리티 기여가 커진다(10초에서 α* tie_set이 [0.3–1.0]까지 확장).

## Q13. "실사용에서 안 되는 케이스는 없나?" (실사용자 테스트 실패 모드 2종, 2026-07-13 실측)

데모 영상 실사용 테스트에서 발견·진단한 실패 모드 2종을 선제 공개한다. 전부 프레임
실물 검증과 코사인 실측을 거쳤다(재현: docs/probes/).

**① 동의어 갭**: 질의 "초밥"이 실재하는 초밥 매대(6:50, 캡션은 "스시"로 명명)를 21위로
놓침 — "스시"로 질의하면 1위. 원인은 임베딩의 어휘 커버리지: cos(초밥,스시)=0.48인데
cos(초밥,김밥)=0.75로 **표면 유사(밥)가 진짜 동의어를 이긴다**(BGE-M3도 동일 실패 —
특정 모델 문제가 아니라 한국어 외래어-고유어 쌍의 학습 공백).
→ **처방 실증**: 사전 기반 질의 확장 + raw 코사인 채널 max 풀링으로 21위→**2위** 복구,
사전 미적중 질의는 top10 완전 불변(부작용 격리). 정식 반영은 dev 검증→test 재평가 절차
필요라 향후 과제(synonym_expansion_probe.py).

**② 언급≠행위 (존재하지 않는 장면의 확신형 오답)**: 질의 "마늘 다지는 장면"에 자막이
"다진 마늘"을 언급하는 세그먼트들이 상위 — 실제 화면은 오이·당근 손질(프레임 확인),
영상에 마늘 다지는 장면 자체가 없음. 임베더는 행위 서술을 언급보다 높게 볼 줄 알지만
(0.85 vs 0.77 실측), **장면이 없으면 언급이 정당하게 1위가 되고 abstention도 통과**한다.
유사도 공간 안에서는 원리적으로 해결 불가 — 재검증 단계(아키텍처)가 필요.
→ **처방 실증**: 3B VLM verifier가 언급 반향 오답 3건 전부 + 사람 기대 라벨이 틀렸던
근접 오답(새우 '손질' 장면)까지 정확히 기각(정밀도 4/4). 단 양성 재현은 2/4 — 기름 속
반죽 새우를 인식 못 함(3B 시각 한계). **재검증 구조는 유효, verifier 품질은 VLM 크기
종속**이 실측으로 확인됨 → 서버급 VLM 확보 후 정식 반영(vlm_verifier_probe.py).

**한계의 3분류 (어떤 한계가 무엇으로 풀리는가):**

| 한계 | 모델 크기로 해소? | 실제 해법 |
|---|---|---|
| 캡션 중국어 드리프트 | 예상됨(7B/8bit) | 서버 GPU 실측 |
| 동의어 갭 | 상관 있으나 비보장(동급 2모델 동일 실패 실측) | 질의 확장(실증됨) |
| 언급≠행위 | 무관(구조 문제) | 프레임 재검증 단계(실증됨, 단 verifier는 크기 종속) |

## Q14. "성능을 더 올릴 여지는? 설계가 최선인가?" (구조 점검 실측, 2026-07-13)

dev 전용 재분석으로 개선 여지를 정량화해 뒀다(전부 test 미접촉, 재현: docs/probes/).
정식 반영은 확정 절차(dev 재탐색→승인→test 재평가)가 필요해 중간발표 후 과제로 관리한다.

**① 유형별 α 분화 — 가장 큰 여유폭.** 단일 α는 상반된 두 유형을 동시에 만족 못 한다:
dev 96 실측에서 유형별 최적 α가 자막형 0.8 / **장면형 0.0** / 복합형 0.4로 극단 분화.
완벽한 유형 분류를 가정한 oracle 조합은 mrr 0.728로 단일 최적(0.655) 대비 **+0.073**,
배포값(α=0.5, 0.630) 대비 +0.098. 장면형은 α=0.5가 α=0 대비 mrr 0.19를 버리는 중.
실현 조건: 질의 유형 분류기(한국어 "~하는 장면" 패턴 등) — 분류 오류가 새 실패 모드가
될 수 있어 신중 검토 대상.

**② minmax 정규화가 구조적으로 유해 — 유의 확인.** per-query minmax는 극값에 민감해
유효 점수 범위를 압축한다. dev 96 융합 방식 비교(쌍체 부트스트랩 CI):

| 융합 | MRR | Δ vs 현행 α=0.5 | 95% CI |
|---|---|---|---|
| 현행 minmax α=0.5 | 0.6296 | — | — |
| **z-score α=0.4** | **0.6941** | **+0.065** | [+0.032, +0.103] 0 배제 |
| 무정규화 raw합 α=0.5 | 0.6741 | +0.045 | [+0.015, +0.081] 0 배제 |
| RRF k=60 (순위 융합) | 0.4465 | −0.183 | 유의 열세 |

**무정규화조차 minmax를 유의하게 이긴다** — 정규화 선택이 성능을 깎고 있었다는 뜻.
RRF 열세는 코사인 크기 정보를 버리면 안 된다는 부정 결과로 기록. ①과 ② 결합 시
추가 상승 여지 있음(미실측).

**③ 영상 군집 균질성 — 우려 점검이 강점으로 반전.** "질의가 영상 안에 군집인데 CI가
i.i.d. 가정"이라는 통계 구조 우려를 점검한 결과, test 4영상(장르 상이)의 mrr 개선이
+0.140/+0.140/+0.153/+0.146으로 **거의 동일** — 개선이 특정 영상·장르 의존이 아니라는
일반화 증거가 질의 수준 CI보다 강하다(영상 jackknife SE 0.002; 군집 4개라 근사치임은 병기).

**④ rep_frame 구조 상한**: 세그먼트당 대표 프레임 1장만 캡셔닝 → 5초 내 장면 전환 시
캡션 채널 재현율에 상한. seg10 ablation에서 캡션 기여 붕괴(α*→1.0)가 방증. 다중 프레임
캡셔닝은 비용 트레이드오프와 함께 향후 과제.

## 한 줄 요약 (오프닝/클로징용)

"큰 수치가 아니라, **주장하는 것만 정확히 검증했다**가 이 연구의 기여다 — dev-only
튜닝·test 접촉 이력 공개·자동 세정·재현 가능한 확정 절차 위에서, 자막 단일 모달리티의
구조적 사각지대(무발화 장면, 질의의 1/3)를 캡션이 메우면서 기존 강점을 거의 해치지
않는 균형점을 소표본에서도 신뢰할 수 있게 찾았다."
